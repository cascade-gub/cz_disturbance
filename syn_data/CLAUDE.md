# CLAUDE.md — Synthetic disturbance–response datasets

Project context for `syn_data/`. Three idealized synthetic time series modeling an
**exogenous climate driver and a lagged response across a regime shift**.
Generated by `make_synthetic.Rmd` (seed `42`, whole-document sequential draws) →
three CSVs. This file documents what those datasets contain and how they were designed
so analysis notebooks can be written against them without re-reading the generator.

**Where this fits.** `syn_data/` is the **idealised leg** of a two-part **methods comparison** —
two ways of quantifying driver→response *synchrony* (peak/moments in analysis I; Morlet wavelet in
analysis II), run here on a clean deterministic system to establish the theory and expectations of
what each method can and cannot see. The **real-data leg**, [`../ms_data/`](../ms_data/) (analyses
III/IV), runs the *same two methods* on real MacroSheds watersheds to test how they hold up under
less-deterministic synchrony. Keep the metric families identical across the two legs so the
comparison stays clean.

## Files

| File | Rows | Scenario |
|------|------|----------|
| `synthetic_shift.csv`  | 4250 | **Shift** — permanent new regime |
| `synthetic_desync.csv` | 4250 | **Desync** — response decouples into noise |
| `synthetic_return.csv` | 4500 | **Return** — transient decoupling into noise, recovers |

`make_synthetic.Rmd` is the generator; `make_synthetic.html` is its knitted output.

## Schema (all three files, identical)

Each row is one time step. Columns:

- `time` — integer step, `1 … N`.
- `climate` — exogenous driver. Clean sine + constant **25%** baseline noise
  (a deliberately noisy driver). Never disturbed directly.
- `response` — the lagged variable, a **lagged log-linear transform of the realized
  (noisy) `climate`**: `log(response) = β·climate(t − lag) + noise`, i.e.
  `response = R0·exp(…)`. Strictly **positive**, right-skewed, with **multiplicative
  (log-space) noise** — a ≈5% baseline term plus a scenario-specific **additional disturbance
  noise**. Because it is built from the realized `climate`, **climate's 25% noise propagates
  (lagged) into `response`**. Analyze in **log space** (`log(response)`) to recover the clean
  lagged-sine structure.
- `phase` — ordered factor, one of `pre` → `disturbance` → `transition` → `post`:
  - `pre` — regime n, before the disturbance
  - `disturbance` — the **single** time step `td` marking the end of regime n
  - `transition` — the adaptation / decoupling window after `td`
  - `post` — the settled terminal regime

`climate`/`response` values are rounded to 6 decimals on export. The generator's internal
`add_climate`/`add_response` noise channels are **not** exported.

## Shared design

**Baseline (all scenarios, regime n):** `climate` is a noisy sine; `log(response)` is a
scaled, lag-shifted copy of that realized `climate` (`response` lags `climate` by 50 steps),
so `climate` and `log(response)` track tightly. In real units `response` is positive and
right-skewed (sharp positive peaks, gentle decays).

Key parameters (from the `params` chunk):

- `P = 500` — period (steps). Lag is a small phase fraction. Since `log(response)` is
  `climate` lag-shifted, `cor(climate, log(response))` at the **matching** lag is ≈ +1; the
  *contemporaneous* relation is `cos(2π·lag/P)` → **+0.81 at lag 50**, **+0.99 at lag 10**,
  attenuated by climate's noise (see Analysis notes).
- `Ac = 1.0` — climate amplitude (constant; the driver, no amplitude blend).
- Regime **n**:   `beta_n=1.0, lag_n=50` (log-space sensitivity 1.0, long lag).
- Regime **n+1**: `beta_p=0.5, lag_p=10` (sensitivity halved, short lag).
- `R0 = 1.0` — baseline scale (`response = R0·exp(…)`; `logR0 = 0` ⇒ `log(response)` centred
  at 0, overlays `climate` directly).
- `lag_max = max(lag_n, lag_p) = 50` — warm-up length; `climate` is generated over the
  extended index `(1−lag_max)…N` so `climate(t − lag)` is always defined.
- `clim_sd = 0.25` — constant 25% iid Gaussian noise on **climate** (linear, noisy driver).
- `base_sd = 0.05` — constant ≈5% baseline noise on **response**, in **log space**
  (multiplicative). Secondary now, since climate's noise dominates the inherited channel.
- `dist_sd = 0.30` — peak additional disturbance noise (30%), **log space**, response only.
- `LEN_REG = 2000` (= 4·P → 4 visible cycles/regime); `LEN_ADAPT = 250` (transition span).

**Morph mechanism.** Transitions use a per-row weight `w ∈ [0,1]` (0 = regime n, 1 = n+1).
The log-signal is the **weighted crossfade of two lagged copies of the realized climate** —
`(1−w)·β_n·climate(t−lag_n) + w·β_p·climate(t−lag_p)` — then `response = R0·exp(logsig +
noise)`. The crossfade is done **in log space** (linear there), so `w` sweeps
sensitivity+lag smoothly and free of *spurious* chirps: both regimes share `P`, so frequency
returns **exactly** to `P` after the transition. The lag change itself is **not**
frequency-flat — shifting the delay `lag_n → lag_p` over a finite window is a Doppler shift,
`f_out = f_in·(1 − dτ/dt)`, a **real transient frequency excursion** of magnitude ≈
`Δlag/window ≈ 16%`. **Only Shift exercises this `w`-morph** (lag 50→10 → one chirp, speeds
up); **Desync and Return hold `w = 0`** (no lag change → no chirp — their transitions are
carried by the signal gain, not a morph). For Shift, frequency-domain analysis (e.g. wavelets)
should read the transition-band chirp as signal, not a numerical artifact. The lag is a plain
index shift into the realized `climate` (no lag-in-sine-argument).

**Disturbance noise.** A separate time-varying Gaussian term added to `log(response)` only
(so multiplicative in real units; climate is never disturbed). Its scale/mean profile differs
per scenario (below).

## Scenario details

### Shift — `synthetic_shift.csv` (N = 4250)
Regime n → disturbance at `td = 2000` → adaptation window `[2001, 2250]` → regime n+1
(permanent). During adaptation, `w` ramps 0→1 (sensitivity/lag blend n→n+1) and additional
response noise decays 30%→0. Terminal state: settled regime n+1 (sensitivity β halved so
`sd(log(response))` ≈ 0.5×, lag 10).
- Phase bands: `pre [1,2000]` · `transition [2001,2250]` · `post [2251,4250]`.

### Desync — `synthetic_desync.csv` (N = 4250)
Regime n → disturbance at `td = 2000` → transition `[2001, 2250]` → regime n+1 is **pure
noise**. The log-signal gain fades 1→0 while additional log-space noise grows, so the
terminal response is **lognormal noise** whose **log**-moments match regime n
(`mu_lr ≈ 0`, `sd_lr ≈ β_n·sd(clag) ≈ 0.75` — the marginal spread of the regime-n
log-signal). The match is **log-space only**: regime n is `exp(sinusoid)` (Bessel-`I₀`) and
the terminal block is `exp(Gaussian)` (lognormal), so at equal log-variance their **raw-unit
variance differs — terminal is ≈ +40%** (`sd` ≈ 1.13 vs 0.94). Compare Desync in log space;
naive linear-variance comparison will mislead. Climate keeps oscillating normally: a
persistent driver the response no longer tracks. `w = 0` throughout — decoupling is carried
entirely by the fading gain, not a morph.
- Phase bands: `pre [1,2000]` · `transition [2001,2250]` · `post [2251,4250]` (noise).

### Return — `synthetic_return.csv` (N = 4500)
A **transient version of Desync**. Regime n → disturbance at `td = 2000` → the response
**degrades into noise**: the signal gain dips 1→0 while additional log-space noise grows to
regime-n matched variance, so at `pk = 2250` the response is momentarily **pure
(matched-variance) noise, decoupled from climate** → then **recovers back to regime n** by
`e1 = 2500`. `w = 0` throughout — **no parameter morph; regime n+1 is never involved**. The
excursion is carried entirely by the signal gain dipping and returning (`gain_r = 1 − tri`),
mirroring Desync but transient. Both ends are regime n.
- Phase bands: `pre [1,2000]` · `transition [2001,2500]` (degrade→noise→return, max
  decoupling at 2250) · `post [2501,4500]` (regime n).

## Analysis notes / gotchas

- **Work in log space.** `log(response)` is a lag-shifted copy of `climate`, so the
  **matched-lag** `cor(climate(t − lag), log(response))` ≈ **+0.99** (regime n: +0.998 at
  lag 50; regime n+1: +0.991 at lag 10 — only `base_sd` breaks the identity), and a
  cross-correlation scan recovers the lag exactly (argmax at 50 pre, 10 post).
  *Contemporaneous* `cor(climate, log(response))` in regime n ≈ **+0.72** (lag-50 phase
  offset), rising to ≈ **+0.87** in regime n+1 (Shift; lag 10). On raw `response` these are
  slightly lower (exp skew; e.g. pre ≈ +0.67). Peak/amplitude/phase-lag analysis should run
  on `log(response)`; `sd(log(response))/sd(climate)` ≈ `β` (≈1.0 pre, ≈0.5 in Shift post).
- `response` is strictly **positive** and right-skewed; its noise is **multiplicative**
  (larger responses noisier). Don't assume Gaussian/additive residuals in linear units — take logs.
- The disturbance is a **single step** (`phase == "disturbance"`, `time == td == 2000`), not
  a span. Grouping by `phase` gives one row for it.
- Only `response` is ever disturbed; `climate` is the same construction across scenarios
  (constant-amplitude noisy sine), differing only through its RNG draws / series length.
- Desync's `post` block is stationary (lognormal) noise — spectral/autocorrelation analysis
  should show climate retaining its P=500 periodicity while response loses it.
- All series are deterministic given seed 42; regenerate by knitting `make_synthetic.Rmd`
  in this directory (CSVs are written to `OUT_DIR = "."`). Old (independent-sine) datasets
  are archived in `pre_loglinear/`.
- **Shared wavelet kernel (analysis_02).** `analysis_02` builds ALL its wavelet helpers (`wco`,
  `band_trace`, `wave_summary`, `plot_*`, the significance helpers, …) from `../R/wavelet_kernel.R`
  via `make_wavelet_kernel(cfg)` + `list2env(..., environment())` — the same kernel
  `ms_data/analysis_04`/`05` use (edit once, not three times). 02's `cfg` is the synthetic leg:
  `P_days = 500` (steps), no phase gating, a `transition` phase, no calendar shading. The red-noise
  cross-power significance mask (Torrence & Compo 1998) is a validation exhibit here: shared power
  towers over red noise at `P≈500` in Shift/Return and dips in Desync — the discrimination the
  saturated coherence cannot make. Deterministic (no RNG).
