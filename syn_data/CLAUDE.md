# CLAUDE.md ‚Äî Synthetic disturbance‚Äìresponse datasets

Project context for `syn_data/`. Three idealized synthetic time series modeling an
**exogenous climate driver and a lagged response across a regime shift**.
Generated by `make_synthetic.Rmd` (seed `42`, whole-document sequential draws) ‚Üí
three CSVs. This file documents what those datasets contain and how they were designed
so analysis notebooks can be written against them without re-reading the generator.

## Files

| File | Rows | Scenario |
|------|------|----------|
| `synthetic_shift.csv`  | 4250 | **Shift** ‚Äî permanent new regime |
| `synthetic_desync.csv` | 4250 | **Desync** ‚Äî response decouples into noise |
| `synthetic_return.csv` | 4500 | **Return** ‚Äî transient excursion, recovers |

`make_synthetic.Rmd` is the generator; `make_synthetic.html` is its knitted output.

## Schema (all three files, identical)

Each row is one time step. Columns:

- `time` ‚Äî integer step, `1 ‚Ä¶ N`.
- `climate` ‚Äî exogenous driver. Clean sine + constant **25%** baseline noise
  (a deliberately noisy driver). Never disturbed directly.
- `response` ‚Äî the lagged variable, a **lagged log-linear transform of the realized
  (noisy) `climate`**: `log(response) = Œ≤¬∑climate(t ‚àí lag) + noise`, i.e.
  `response = R0¬∑exp(‚Ä¶)`. Strictly **positive**, right-skewed, with **multiplicative
  (log-space) noise** ‚Äî a ‚âà5% baseline term plus a scenario-specific **additional disturbance
  noise**. Because it is built from the realized `climate`, **climate's 25% noise propagates
  (lagged) into `response`**. Analyze in **log space** (`log(response)`) to recover the clean
  lagged-sine structure.
- `phase` ‚Äî ordered factor, one of `pre` ‚Üí `disturbance` ‚Üí `transition` ‚Üí `post`:
  - `pre` ‚Äî regime n, before the disturbance
  - `disturbance` ‚Äî the **single** time step `td` marking the end of regime n
  - `transition` ‚Äî the adaptation / decoupling / excursion window after `td`
  - `post` ‚Äî the settled terminal regime

`climate`/`response` values are rounded to 6 decimals on export. The generator's internal
`add_climate`/`add_response` noise channels are **not** exported.

## Shared design

**Baseline (all scenarios, regime n):** `climate` is a noisy sine; `log(response)` is a
scaled, lag-shifted copy of that realized `climate` (`response` lags `climate` by 50 steps),
so `climate` and `log(response)` track tightly. In real units `response` is positive and
right-skewed (sharp positive peaks, gentle decays).

Key parameters (from the `params` chunk):

- `P = 500` ‚Äî period (steps). Lag is a small phase fraction. Since `log(response)` is
  `climate` lag-shifted, `cor(climate, log(response))` at the **matching** lag is ‚âà +1; the
  *contemporaneous* relation is `cos(2œÄ¬∑lag/P)` ‚Üí **+0.81 at lag 50**, **+0.99 at lag 10**,
  attenuated by climate's noise (see Analysis notes).
- `Ac = 1.0` ‚Äî climate amplitude (constant; the driver, no amplitude blend).
- Regime **n**:   `beta_n=1.0, lag_n=50` (log-space sensitivity 1.0, long lag).
- Regime **n+1**: `beta_p=0.5, lag_p=10` (sensitivity halved, short lag).
- `R0 = 1.0` ‚Äî baseline scale (`response = R0¬∑exp(‚Ä¶)`; `logR0 = 0` ‚áí `log(response)` centred
  at 0, overlays `climate` directly).
- `lag_max = max(lag_n, lag_p) = 50` ‚Äî warm-up length; `climate` is generated over the
  extended index `(1‚àílag_max)‚Ä¶N` so `climate(t ‚àí lag)` is always defined.
- `clim_sd = 0.25` ‚Äî constant 25% iid Gaussian noise on **climate** (linear, noisy driver).
- `base_sd = 0.05` ‚Äî constant ‚âà5% baseline noise on **response**, in **log space**
  (multiplicative). Secondary now, since climate's noise dominates the inherited channel.
- `dist_sd = 0.30` ‚Äî peak additional disturbance noise (30%), **log space**, response only.
- `LEN_REG = 2000` (= 4¬∑P ‚Üí 4 visible cycles/regime); `LEN_ADAPT = 250` (transition span).

**Morph mechanism.** Transitions use a per-row weight `w ‚àà [0,1]` (0 = regime n, 1 = n+1).
The log-signal is the **weighted crossfade of two lagged copies of the realized climate** ‚Äî
`(1‚àíw)¬∑Œ≤_n¬∑climate(t‚àílag_n) + w¬∑Œ≤_p¬∑climate(t‚àílag_p)` ‚Äî then `response = R0¬∑exp(logsig +
noise)`. The crossfade is done **in log space** (linear there), so `w` sweeps
sensitivity+lag smoothly and free of *spurious* chirps: both regimes share `P`, so frequency
returns **exactly** to `P` after the transition. The lag change itself is **not**
frequency-flat ‚Äî shifting the delay `lag_n ‚Üí lag_p` over a finite window is a Doppler shift,
`f_out = f_in¬∑(1 ‚àí dœÑ/dt)`, a **real transient frequency excursion** of magnitude ‚âà
`Œîlag/window ‚âà 16%` (Shift: one excursion, speeds up; Return: two, out then back; Desync:
none, `w=0`). This is the kinematic signature of the lag change, not a numerical artifact ‚Äî
frequency-domain analysis (e.g. wavelets) should read the transition-band feature as signal.
The lag is a plain index shift into the realized `climate` (no lag-in-sine-argument).

**Disturbance noise.** A separate time-varying Gaussian term added to `log(response)` only
(so multiplicative in real units; climate is never disturbed). Its scale/mean profile differs
per scenario (below).

## Scenario details

### Shift ‚Äî `synthetic_shift.csv` (N = 4250)
Regime n ‚Üí disturbance at `td = 2000` ‚Üí adaptation window `[2001, 2250]` ‚Üí regime n+1
(permanent). During adaptation, `w` ramps 0‚Üí1 (sensitivity/lag blend n‚Üín+1) and additional
response noise decays 30%‚Üí0. Terminal state: settled regime n+1 (sensitivity Œ≤ halved so
`sd(log(response))` ‚âà 0.5√ó, lag 10).
- Phase bands: `pre [1,2000]` ¬∑ `transition [2001,2250]` ¬∑ `post [2251,4250]`.

### Desync ‚Äî `synthetic_desync.csv` (N = 4250)
Regime n ‚Üí disturbance at `td = 2000` ‚Üí transition `[2001, 2250]` ‚Üí regime n+1 is **pure
noise**. The log-signal gain fades 1‚Üí0 while additional log-space noise grows, so the
terminal response is **lognormal noise** whose **log**-moments match regime n
(`mu_lr ‚âà 0`, `sd_lr ‚âà Œ≤_n¬∑sd(clag) ‚âà 0.75` ‚Äî the marginal spread of the regime-n
log-signal). The match is **log-space only**: regime n is `exp(sinusoid)` (Bessel-`I‚ÇÄ`) and
the terminal block is `exp(Gaussian)` (lognormal), so at equal log-variance their **raw-unit
variance differs ‚Äî terminal is ‚âà +40%** (`sd` ‚âà 1.13 vs 0.94). Compare Desync in log space;
naive linear-variance comparison will mislead. Climate keeps oscillating normally: a
persistent driver the response no longer tracks. `w = 0` throughout ‚Äî decoupling is carried
entirely by the fading gain, not a morph.
- Phase bands: `pre [1,2000]` ¬∑ `transition [2001,2250]` ¬∑ `post [2251,4250]` (noise).

### Return ‚Äî `synthetic_return.csv` (N = 4500)
Regime n ‚Üí disturbance at `td = 2000` ‚Üí single excursion toward n+1 that **peaks at
`pk = 2250` then decays straight back to regime n** by `e1 = 2500`. No sustained n+1: `w`
is triangular (0‚Üí1 at peak‚Üí0). Additional response noise 30% at the disturbance ‚Üí 0 by
re-settle. Both ends are regime n.
- Phase bands: `pre [1,2000]` ¬∑ `transition [2001,2500]` (excursion, peak at 2250) ¬∑
  `post [2501,4500]` (back to regime n).

## Analysis notes / gotchas

- **Work in log space.** `log(response)` is a lag-shifted copy of `climate`, so the
  **matched-lag** `cor(climate(t ‚àí lag), log(response))` ‚âà **+0.99** (regime n: +0.998 at
  lag 50; regime n+1: +0.991 at lag 10 ‚Äî only `base_sd` breaks the identity), and a
  cross-correlation scan recovers the lag exactly (argmax at 50 pre, 10 post).
  *Contemporaneous* `cor(climate, log(response))` in regime n ‚âà **+0.72** (lag-50 phase
  offset), rising to ‚âà **+0.87** in regime n+1 (Shift; lag 10). On raw `response` these are
  slightly lower (exp skew; e.g. pre ‚âà +0.67). Peak/amplitude/phase-lag analysis should run
  on `log(response)`; `sd(log(response))/sd(climate)` ‚âà `Œ≤` (‚âà1.0 pre, ‚âà0.5 in Shift post).
- `response` is strictly **positive** and right-skewed; its noise is **multiplicative**
  (larger responses noisier). Don't assume Gaussian/additive residuals in linear units ‚Äî take logs.
- The disturbance is a **single step** (`phase == "disturbance"`, `time == td == 2000`), not
  a span. Grouping by `phase` gives one row for it.
- Only `response` is ever disturbed; `climate` is the same construction across scenarios
  (constant-amplitude noisy sine), differing only through its RNG draws / series length.
- Desync's `post` block is stationary (lognormal) noise ‚Äî spectral/autocorrelation analysis
  should show climate retaining its P=500 periodicity while response loses it.
- All series are deterministic given seed 42; regenerate by knitting `make_synthetic.Rmd`
  in this directory (CSVs are written to `OUT_DIR = "."`). Old (independent-sine) datasets
  are archived in `pre_loglinear/`.
